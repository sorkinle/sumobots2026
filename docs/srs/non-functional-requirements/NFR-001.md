# NFR-001: Input-to-Output Latency Budget

## Requirement

**Category:** Performance
**Statement:** The firmware shall maintain a **95th-percentile** end-to-end latency from BLE-HID report receipt to motor PWM-duty update of ≤ **50 ms**, and a **99th-percentile** of ≤ **100 ms**, under nominal RF conditions (≤ 3 m line-of-sight) and ≤ 80% CPU load.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.1 | Translate each controller input into a motor-output update within a single perceptual round-trip |
| Applies To FRs | FR-001, FR-002, FR-003 | Loop rate, per-event latency, throttle pass-through |
| Customer Problem | CP.1 | Real-time control with imperceptible input lag |

## Acceptance Criteria

- [ ] Instrumented build emits per-event timestamps; 5-minute drive run yields p95 ≤ 50 ms
- [ ] Same run yields p99 ≤ 100 ms
- [ ] No single event > 250 ms (would imply link-loss should have triggered per FR-006)

## Measurement Method

Build with `LATENCY_TRACE=1`. Firmware records `t_hid_received` and `t_pwm_updated` for each event and emits both over USB-serial. Post-process with the analyzer script (TBD) to produce a histogram and percentile summary.
