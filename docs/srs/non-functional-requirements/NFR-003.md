# NFR-003: ARMED-Gate Safety Invariant

## Requirement

**Category:** Safety
**Statement:** The motor PWM output path shall contain exactly **one** architectural check — `state == ARMED` — that forces motor output to 0 when the State Machine is not in ARMED. This check shall be present in every code path that writes motor PWM duty, and its presence shall be verifiable by source-code review and by automated test.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.7 | Keep motor outputs forced off whenever the robot is not in ARMED |
| Applies To FRs | FR-010 | DISARMED motor-off invariant |
| Customer Problem | CP.3 | Safe stop on command / link loss |
| Customer Problem | CP.5 | Protect hardware from foreseeable failure modes |

## Acceptance Criteria

- [ ] Code review: there is exactly one `state == ARMED` guard immediately preceding every PWM duty write (verifiable by grep or static check)
- [ ] No code path writes PWM duty bypassing the guard
- [ ] An automated test forces each non-ARMED state, injects full-throttle inputs, and asserts that measured PWM duty = 0
- [ ] Mutating the guard (e.g., commenting it out) causes the automated test to fail

## Measurement Method

Source-code grep for PWM-write call sites + manual review + automated state-injection test in the test harness.
