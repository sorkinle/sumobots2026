# NFR-006: Firmware Flash Footprint

## Requirement

**Category:** Resource
**Statement:** The compiled release firmware image shall fit within **1 MB** of ESP32 flash, leaving headroom for a future OTA-update partition layout and a separate NVS / configuration partition.

**Priority:** Should Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | — | (indirect — no direct CN; supports future CP.5 recovery features such as OTA rollback) |
| Applies To FRs | (all) | All FR implementations contribute to the image size |

## Acceptance Criteria

- [ ] Release build (`-Os` or `-O2`) reports total flash usage ≤ 1 MB in build output
- [ ] No mandatory third-party dependency (e.g., Bluepad32) is excluded to meet the budget

## Measurement Method

Read image-size report from `esptool.py`, the PlatformIO build summary, or the Arduino-IDE build output after a release build.
