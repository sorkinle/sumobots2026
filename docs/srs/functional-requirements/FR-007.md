# FR-007: Initial Pairing in PAIRING Mode

## Requirement

**Statement:** On boot, if no controller BLE address is stored in NVS, the firmware shall enter **PAIRING** mode and accept connection from the first Xbox Series controller that advertises BLE-HID services. On successful pairing, the firmware shall persist the controller's BLE address to NVS and transition to **DISARMED**.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.5 | Establish and re-establish a BLE-HID connection automatically |
| Customer Problem | CP.4 | The robot must maintain a reliable wireless link to the Xbox controller |

## Acceptance Criteria

- [ ] Fresh boot with cleared NVS → PAIRING mode entered within 5 s of power-on
- [ ] Status LED shows PAIRING pattern
- [ ] First Xbox controller put into pairing range connects within 30 s
- [ ] After successful pair, controller BLE address persists across a power cycle (verified by `dump`)
- [ ] After successful pair, State Machine transitions to DISARMED (not directly to ARMED)

## Notes

The "first controller wins" policy keeps initial setup simple. The `forget` console command (part of FR-016) clears the NVS entry to re-pair with a different controller.
