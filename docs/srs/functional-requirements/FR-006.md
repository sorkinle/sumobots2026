# FR-006: Link-Loss Failsafe

## Requirement

**Statement:** If the firmware does not receive a new HID input report from the paired controller for longer than **250 ms**, it shall set motor PWM duty to 0 on both sides and transition the State Machine to **DISARMED**.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.4 | Drive motor outputs to zero automatically when the controller link has been silent for longer than a bounded timeout |
| Customer Problem | CP.3 | The robot must stop immediately on operator command or link loss |

## Acceptance Criteria

- [ ] With robot ARMED and motors commanded, powering down or shielding the controller causes measured PWM duty = 0 within 300 ms (250 ms target + 50 ms detection grace)
- [ ] After link loss, status LED transitions through DISARMED to PAIRING (per FR-008 auto-reconnect)
- [ ] During 60 s of sustained driving with stable RF, zero spurious link-loss triggers
- [ ] The 250 ms timeout is configurable via the `linkloss_timeout_ms` parameter (default 250)

## Notes

The 250 ms budget is well below human reaction time and accommodates typical BLE-HID report intervals (8–16 ms nominal). Setting `linkloss_timeout_ms` shorter risks false trips in noisy RF; longer risks unsafe runaway after a real dropout.
