# FR-010: DISARMED Motor-Off Invariant

## Requirement

**Statement:** The firmware shall ensure that motor PWM duty is exactly 0 on both sides whenever the State Machine is in **any state other than ARMED** (i.e., BOOT, PAIRING, DISARMED, or FAULT), regardless of any controller input being received or any tunable parameter value.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.7 | Keep motor outputs forced off whenever the robot is not in ARMED |
| Customer Problem | CP.5 | The robot must protect motors, drivers, and battery from foreseeable failure modes |

## Acceptance Criteria

- [ ] In DISARMED, full-throttle stick input produces measured PWM duty = 0 on both sides
- [ ] In FAULT, full-throttle stick input produces measured PWM duty = 0
- [ ] In PAIRING, no controller input is even possible, but bench-injected signals produce PWM duty = 0
- [ ] In BOOT, PWM peripheral is either uninitialized or driving 0% duty before any input is sampled
- [ ] Source-code review: exactly **one** `state == ARMED` guard exists immediately preceding every PWM duty write
- [ ] Static check or unit test fails if the guard is removed or bypassed

## Notes

This is **the** single most important safety invariant in the firmware. NFR-003 enforces the architectural form (one guard, code-review verifiable).
