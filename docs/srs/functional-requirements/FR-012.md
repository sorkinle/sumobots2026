# FR-012: Center Deadband

## Requirement

**Statement:** The firmware shall apply a center deadband to all controller stick axes prior to mixing. The deadband half-width is configurable via the `deadband` parameter (range 0.00 – 0.30, default 0.10). For raw stick magnitude `|x| ≤ deadband`, the post-deadband value is 0; for `|x| > deadband`, the value is rescaled linearly to span the full [0, 1] range.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.9 | Apply input shaping (deadband + response curve) for predictable feel |
| Customer Problem | CP.6 | Intuitive tank-style controls that map predictably into motion |

## Acceptance Criteria

- [ ] With `deadband = 0.10`, a measured stick deflection ≤ 0.10 produces 0 mixed output on both sides
- [ ] Stick deflection > 0.10 produces a continuous monotonic mapping starting from 0
- [ ] Idle (untouched) stick produces 0 PWM duty for a 60 s bench test (no jitter past deadband)
- [ ] `set deadband 0.0` disables deadband entirely (raw input passes through)
- [ ] Out-of-range values (e.g., `set deadband 0.5`) are rejected

## Notes

The deadband is applied **before** the response curve (FR-013) and before mapping (FR-011). This ordering makes "idle stick drift = no motion" a guaranteed property regardless of curve/mapping choices.
