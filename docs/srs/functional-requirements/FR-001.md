# FR-001: Input Sample-and-Update Loop Rate

## Requirement

**Statement:** The firmware shall execute its input-sample → mix → motor-output cycle at a frequency of at least **100 Hz** (per-cycle period ≤ 10 ms) under nominal load.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.1 | Translate each controller input into a motor-output update within a single perceptual round-trip |
| Customer Problem | CP.1 | The operator must drive the robot in real time with imperceptible input lag |

## Acceptance Criteria

- [ ] Instrumented build logs cycle timestamps; ≥99% of cycles measured ≤ 10 ms over a 60 s run
- [ ] Maximum observed cycle period ≤ 20 ms (2× target) over the 60 s run
- [ ] No motor-output stalls > 50 ms observed in the run

## Notes

This is the **loop-level** latency budget. FR-002 covers the **per-event** latency budget (HID report → PWM update). The two are complementary: FR-001 ensures the firmware is always making forward progress; FR-002 ensures any specific input is acted on promptly.
