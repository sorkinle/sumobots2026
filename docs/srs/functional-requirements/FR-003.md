# FR-003: Max-Throttle Pass-Through

## Requirement

**Statement:** When the operator commands full throttle (post-deadband, post-curve input magnitude = 1.0), the firmware shall produce a per-side PWM duty equal to the configured `max_power` parameter, with no further firmware-imposed attenuation.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.2 | Deliver full available per-side motor power with no firmware-imposed cap below configured maximum |
| Customer Problem | CP.2 | The robot must deliver full motor torque to both wheels on a side when commanded |

## Acceptance Criteria

- [ ] With `max_power = 1.0` and full forward stick, measured PWM duty ≥ 99% on both sides
- [ ] With `max_power = 0.75` and full forward stick, measured PWM duty between 73% and 77% on both sides
- [ ] No latent slew or curve causes a steady-state duty below `max_power` when input is held > 2 s
- [ ] Reverse direction reaches the same magnitude (within 2 percentage points) as forward

## Notes

The parameter `max_power` is a safety/tuning knob (lets the pilot dial in less-than-full power during tuning). It must never be silently overridden by other logic.
