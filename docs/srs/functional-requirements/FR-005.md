# FR-005: Arming Combination

## Requirement

**Statement:** The firmware shall transition from **DISARMED** to **ARMED** only when the operator holds the Xbox **LB + RB + A** buttons simultaneously and continuously for at least **1.0 seconds**. Releasing any of the three buttons before the 1.0 s mark shall abort the arming action with no state change.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.15 | Require a deliberate, multi-button action (unlikely to occur accidentally) to enter ARMED |
| Customer Problem | CP.3 | Safe stop / safe state |
| Customer Problem | CP.5 | Protect hardware from accidental power-on |

## Acceptance Criteria

- [ ] Holding LB + RB + A together for ≥ 1.0 s in DISARMED → transitions to ARMED
- [ ] Releasing any of the three buttons before 1.0 s → State Machine remains DISARMED, motors=0
- [ ] Holding the combo while in FAULT, PAIRING, or BOOT → no state change (arming is only reachable from DISARMED)
- [ ] On entering ARMED, status LED switches to the ARMED pattern within 1 s

## Notes

The three-button combo is chosen because it requires deliberate finger placement; no two of those buttons is a common idle hand position. The 1.0 s hold filters out brief accidental contacts. Both choices are tunable via the `arm_combo` and `arm_hold_ms` parameters (see FR-016) if a different bias is desired.
