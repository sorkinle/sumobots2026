# FR-002: HID-Report → PWM Latency

## Requirement

**Statement:** The firmware shall update motor PWM duty within **50 ms** of receiving any new HID input report from the paired controller, measured from BLE-stack receipt to PWM peripheral write.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.1 | Translate each controller input into a motor-output update within a single perceptual round-trip |
| Customer Problem | CP.1 | The operator must drive the robot in real time with imperceptible input lag |

## Acceptance Criteria

- [ ] Instrumented build records HID-receipt timestamps and PWM-update timestamps
- [ ] 95th-percentile end-to-end latency ≤ 50 ms over a 60 s drive test
- [ ] 99th-percentile end-to-end latency ≤ 100 ms over the same test
- [ ] No single event > 250 ms (would imply link-loss failsafe should have triggered — see FR-006)

## Notes

Pair with **NFR-001** (the same number in quality-attribute form). FR-002 is the behavior; NFR-001 is the verified percentile budget.
