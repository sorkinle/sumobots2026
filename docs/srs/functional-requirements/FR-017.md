# FR-017: NVS Persistence

## Requirement

**Statement:** Upon receiving the `save` command, the firmware shall persist all tunable parameters and the paired-controller BLE address to ESP32 NVS using a versioned key. On boot, the firmware shall read these values from NVS and apply them. If NVS contains no values, or values from an incompatible schema version, the firmware shall apply documented defaults and proceed without error.

**Priority:** Should Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.13 | Persist tuned parameters across reboots in non-volatile storage |
| Customer Problem | CP.8 | Tune driving parameters without recompiling and re-flashing |

## Acceptance Criteria

- [ ] After `save`, power-cycling the firmware produces the saved values (verified by `dump`)
- [ ] Erasing NVS and rebooting produces documented default values (verified by `dump`) with no boot-time error
- [ ] `save` completes within 1 s of command receipt
- [ ] A future firmware version with an incompatible schema version reads NVS, falls back to defaults, and logs a one-line notice — without crashing

## Notes

The Configuration Store is the **only** firmware component that writes NVS (per the architectural decisions in Software Vision). This bounds the failure surface for NVS corruption.
