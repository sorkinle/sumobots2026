# FR-015: On-Demand Status Snapshot

## Requirement

**Statement:** Upon receiving the text command `status` (followed by newline) on the USB-serial console, the firmware shall emit, within **100 ms**, a single-line snapshot containing the current operating mode, controller link state, filtered battery voltage, active fault flags, the most recent input vector (per-axis post-deadband+curve values), and the most recent output vector (per-side PWM duty).

**Priority:** Should Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.11 | Emit a state snapshot over USB-serial on request |
| Customer Problem | CP.7 | The operator expects clear feedback on robot state |
| Customer Problem | CP.9 | The operator hopes for diagnostic telemetry for post-match review |

## Acceptance Criteria

- [ ] Typing `status<enter>` at the serial console produces a single line matching the documented format
- [ ] All fields are populated with current values as of the moment of the command
- [ ] Snapshot is emitted within 100 ms of command receipt
- [ ] Snapshot output does not interleave with periodic telemetry mid-line

## Notes

The snapshot format is fixed and machine-parseable (key=value pairs separated by spaces) so the same output can feed both a human eye and a scripted log analyzer.
