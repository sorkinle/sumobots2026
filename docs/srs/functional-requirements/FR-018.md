# FR-018: Periodic Telemetry Stream

## Requirement

**Statement:** The firmware shall emit a periodic telemetry record over USB-serial at the rate set by the `telemetry_rate` parameter (range 0 – 20 Hz, default 5 Hz). Each record shall be a single line containing the current operating mode, controller link state, filtered battery voltage, active fault flags, current input vector, and current output vector. `telemetry_rate = 0` disables the stream.

**Priority:** Should Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.14 | Emit a periodic telemetry stream over USB-serial at a configurable rate |
| Customer Problem | CP.9 | The operator hopes the firmware emits diagnostic telemetry for post-match review |

## Acceptance Criteria

- [ ] With `telemetry_rate = 5`, exactly 5 records per second are emitted (± 10%)
- [ ] With `telemetry_rate = 20`, exactly 20 records per second are emitted (± 10%)
- [ ] With `telemetry_rate = 0`, no records are emitted
- [ ] Telemetry emission does not violate the FR-002 latency budget (no input-loop stall > 50 ms attributable to telemetry)
- [ ] Telemetry record format is identical to the FR-015 snapshot format (same field set, same parser-friendly key=value layout)

## Notes

The record format intentionally matches the on-demand snapshot from FR-015, so the same parser handles both streams.
