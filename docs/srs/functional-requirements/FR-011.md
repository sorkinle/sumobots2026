# FR-011: Input Mapping Mode Selector

## Requirement

**Statement:** The firmware shall support at least two input-mapping modes, selected at runtime by the `mapping_mode` parameter:

- **`arcade`** — Left stick **Y** axis sets the forward/reverse magnitude applied symmetrically to both sides; left stick **X** axis biases the left/right magnitudes (single-stick driving).
- **`tank`** — Left stick **Y** axis sets the left-side wheel command; right stick **Y** axis sets the right-side wheel command (dual-stick differential driving).

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.8 | Selectable arcade or tank mapping |
| Customer Problem | CP.6 | Intuitive tank-style controls that map predictably into motion |

## Acceptance Criteria

- [ ] `set mapping_mode arcade` → left stick Y produces symmetric forward/reverse on both sides; left stick X produces a left/right bias
- [ ] `set mapping_mode tank` → left and right sticks Y produce independent per-side commands
- [ ] Switching modes at runtime takes effect within one input loop cycle (≤ 10 ms)
- [ ] Invalid mode values produce an error and leave the current mode unchanged

## Notes

Adding additional mapping modes (e.g., trigger-based "drive trigger / steer stick") is a future extension. The parameter accepts only the documented values for v1.
