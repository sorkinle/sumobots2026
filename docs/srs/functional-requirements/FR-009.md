# FR-009: Brownout Detection

## Requirement

**Statement:** The firmware shall sample battery pack voltage at ≥ **10 Hz**, apply a low-pass filter with a configurable time constant (`vbatt_filter_ms`, default 200 ms), and transition the State Machine to **FAULT** mode (with motor PWM duty set to 0) if the filtered voltage stays below the configured `vbatt_threshold` parameter for ≥ **500 ms**.

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.6 | Drive motors off and enter fault when battery voltage falls below safe threshold |
| Customer Problem | CP.5 | The robot must protect motors, drivers, and battery from electrical/thermal failure modes |

## Acceptance Criteria

- [ ] Bench test with adjustable supply: dropping Vbatt below `vbatt_threshold` for 600 ms → State Machine transitions to FAULT and measured PWM duty = 0
- [ ] Brief sag (≤ 200 ms) below `vbatt_threshold` does **not** trigger FAULT (filter rejects transients)
- [ ] On entering FAULT, status LED transitions to the FAULT pattern within 1 s
- [ ] Default `vbatt_threshold` for a 4S LiPo is 13.2 V (3.3 V/cell) and is overridable via the console (FR-016)

## Notes

The threshold default trades capacity utilization for cell safety. Pilots wanting more runtime can drop to 12.8 V (3.2 V/cell) at the cost of pushing the LiPo closer to damage. The filter prevents inrush-current dips from causing nuisance trips.
