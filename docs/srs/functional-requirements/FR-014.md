# FR-014: Status LED Patterns

## Requirement

**Statement:** The firmware shall drive the on-board status LED with the following distinct patterns, one per State Machine mode:

| Mode | LED Pattern |
|------|-------------|
| **BOOT** (first ~1 s) | Off |
| **PAIRING** | Fast blink at ~5 Hz, 50% duty |
| **DISARMED** | Slow blink at ~1 Hz, 25% duty (short on, long off) |
| **ARMED** | Solid on |
| **FAULT** | Triple-blink burst (3× rapid on/off) followed by 1 s off, repeating |

**Priority:** Must Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.10 | Display current operating mode on the status LED using distinct patterns |
| Customer Problem | CP.7 | The operator expects clear local feedback on the robot's state |

## Acceptance Criteria

- [ ] All five patterns are implemented and visually distinct
- [ ] Mode change → new LED pattern visible within 1 s
- [ ] BOOT pattern (LED off) is observed for ≥ 500 ms after power-on, so a stuck-on LED at power-on visibly indicates a hardware fault
- [ ] LED behavior is independent of motor command (LED keeps blinking correctly even at full throttle)

## Notes

Patterns must be distinguishable per **NFR-005** (≥1 m, naive observer, 4-out-of-4 blind test).
