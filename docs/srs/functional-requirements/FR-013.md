# FR-013: Response Curve

## Requirement

**Statement:** The firmware shall apply a configurable response curve to mixed per-side output before motor PWM duty assignment. Supported curves are selected by `curve_type`:

- **`linear`** — output equals input
- **`expo`** — exponential softening of small inputs (smaller inputs produce proportionally smaller outputs; full input still reaches full output), with aggressiveness set by `curve_factor` (range 0.0 – 1.0, where 0.0 = linear and 1.0 = maximally exponential)

**Priority:** Should Have
**Status:** Draft

## Traceability

| Traces To | ID | Description |
|-----------|-----|-------------|
| Customer Need | CN.9 | Apply input shaping (deadband + response curve) for predictable feel |
| Customer Problem | CP.6 | Intuitive tank-style controls that map predictably into motion |

## Acceptance Criteria

- [ ] `set curve_type linear` → output magnitude equals input magnitude (within 1%)
- [ ] `set curve_type expo, curve_factor 0.7` → input of 0.5 produces output strictly less than 0.5; input of 1.0 produces output ≥ 0.99
- [ ] Switching curve type at runtime takes effect within one input loop cycle
- [ ] Invalid `curve_type` values are rejected with a console error and current curve is preserved

## Notes

The expo curve helps fine maneuvering at low throttle without sacrificing full pushing power at full throttle — useful for the heavy 4 kg, high-traction-wheel setup.
